# Phase 5 — Exports

Implements PDF and Excel export generation for both summary (anonymized)
and executor (full detail) report types. Executor exports require
re-authentication before download.

## Export types and anonymization

| export_type      | Account numbers    | Institution names | Member names | Values |
| ---------------- | ------------------ | ----------------- | ------------ | ------ |
| `pdf_summary`    | Last 4 digits only | Shown             | Shown        | Shown  |
| `pdf_executor`   | Full (decrypted)   | Full (decrypted)  | Full         | Shown  |
| `excel_summary`  | Last 4 digits only | Shown             | Shown        | Shown  |
| `excel_executor` | Full (decrypted)   | Full (decrypted)  | Full         | Shown  |

Anonymization is a parameter, not a format distinction. The same generator
handles both modes; the `anonymized` flag controls whether encrypted fields
are decrypted fully or masked.

---

## Re-authentication gate

Executor exports (`pdf_executor`, `excel_executor`) require a `reauth_token`
in the request header. This is obtained by `POST /api/v1/auth/reauth`
(see Phase 1), which asks the user to confirm their password and returns a
10-minute single-use token.

The export endpoint validates the `reauth_token` before enqueuing the job.
If the token is missing, expired, or already used, return 403 with
`"detail": "Executor export requires re-authentication"`.

The `reauth_token` is single-use: once used to authorize an export job,
it is invalidated in Redis (store a `reauth_used:{token_hash}` key with 10-min TTL).

Only `primary` role users may generate executor exports. Partners attempting
executor exports receive 403 regardless of reauth token.

## PDF export structure

### PDF summary

Generated with WeasyPrint from an HTML template.

1. **Cover page**: "HearthLedger Financial Summary" | Household name |
   Report date | Date range covered
2. **Net worth snapshot**: Total assets, total liabilities, net worth.
   Table of accounts: nickname, type, owner, account number (last 4),
   institution, balance as of date.
3. **Cash flow summary**: Income vs expenses table by month. Savings rate.
4. **Spending by category**: Top 10 categories with amounts and percentages.
5. **Budget vs actuals** (if budgets exist): Category, budget, actual, variance.
6. **Investment & retirement summary**: Account list with balances and
   YTD contributions.

### PDF executor

All sections from summary, plus:

1. **Full account directory**: Institution name (decrypted), full account
   number (decrypted), routing number (decrypted if present), account type,
   owner member name, current balance, notes (decrypted).
2. **Real estate holdings**: Property address (decrypted), purchase date,
   purchase price, current estimated value, linked mortgage account.
3. **Debt schedule**: Full account detail, current balance, interest rate,
   minimum payment, estimated payoff date.
4. **FIRE scenario snapshot** (if scenarios exist): scenario name, FIRE number,
   projected FIRE year, income streams.
5. **Audit summary page**: "This report was generated by [member name] on
   [datetime]. Last account modification: [date]. Report covers [date range]."

## Excel export structure

All workbooks use `openpyxl`. Column headers are bold. Monetary values use
`$#,##0.00` number format. Dates use `YYYY-MM-DD`. Tables use alternating
row shading.

### Sheets (both summary and executor)

1. **Net Worth History** — monthly series: Date, Total Assets, Total Liabilities,
   Net Worth, with breakdown columns by account type.
2. **Account Directory** — one row per account. Summary: nickname, type, owner,
   last-4, institution, balance. Executor: full account number, institution,
   routing, notes.
3. **Transactions** — all transactions in date range. Columns: Date, Account,
   Payee, Category, Amount, Property (if tagged), Tags, Reviewed.
   Auto-filter enabled on all columns.
4. **Budget vs Actuals** — category, budget, actual, remaining, % used,
   one column per month in range.
5. **Spending by Category** — category, total amount, % of total, transaction count.
6. **Debt Schedule** — account, institution, balance, rate, minimum payment,
   payoff date (avalanche). Executor adds full account numbers.
7. **FIRE Projections** — one row per year: Year, Age, Portfolio Value,
   Annual Income, Annual Spend, Annual Savings, FIRE Number, FIRE Reached.

## API endpoints — Phase 5

### Export

```
POST /api/v1/exports
  body: {
    export_type: "pdf_summary" | "pdf_executor" | "excel_summary" | "excel_executor",
    from_date: "YYYY-MM-DD",
    to_date: "YYYY-MM-DD",
    account_ids?: [uuid],   // null = all visible accounts
    include_transactions?: bool  // default true for executor, false for summary
  }
  headers (executor only): X-Reauth-Token: <reauth_token>
  requires: primary or partner (executor: primary only)
  returns: { export_job_id: uuid }

GET /api/v1/exports/{id}
  returns: { status, export_type, created_at, completed_at, filename? }

GET /api/v1/exports/{id}/download
  streams the generated file as an attachment
  content-type: application/pdf or
                application/vnd.openxmlformats-officedocument.spreadsheetml.sheet
  content-disposition: attachment; filename="hearthledger_summary_2025-01-15.pdf"

GET /api/v1/exports
  list recent export jobs for household (last 30)
```

### ARQ export task

```python
# backend/app/worker/tasks/export_tasks.py

async def generate_export(ctx, export_job_id: str) -> None:
    job = await get_export_job(export_job_id)
    await update_job_status(job, "processing")
    try:
        if job.export_type.startswith("pdf"):
            path = await pdf_exporter.generate(job)
        else:
            path = await excel_exporter.generate(job)
        await update_job_status(job, "complete", filename=path)
        await audit_repo.write(ctx=system_ctx, action="export.generated",
                               entity_type="export_job", entity_id=job.id,
                               new_value={"export_type": job.export_type,
                                          "anonymized": job.anonymized})
    except Exception as e:
        await update_job_status(job, "failed", error_message=str(e))
        await audit_repo.write(...)
```

## Frontend — Phase 5

### Export modal

Triggered by "Export" button in the top nav. Two-step:

**Step 1 — Configure:**

- Export format selector (four cards: PDF Summary, PDF Full/Executor,
  Excel Summary, Excel Full/Executor). Executor options are grayed out
  and labeled "Primary members only" for partner users.
- Date range picker (from / to)
- Account selector (all or specific accounts)

**Step 2 (executor only) — Re-authenticate:**

- Password confirmation prompt: "Executor reports contain full account
  numbers and sensitive financial details. Please confirm your password
  to continue."
- On success: fetches `reauth_token`, proceeds to generate.
- On failure: shows error, returns to password prompt (max 3 attempts).

**Step 3 — Generating:**

- "Generating your report..." with spinner.
- Polls `GET /api/v1/exports/{id}` every 2 seconds.
- On complete: "Your report is ready" with Download button.
- On failed: error message with retry option.

### `/settings/exports`

Table of recent exports: type badge, date range, generated by, generated at,
Download button (re-fetches file). Executor exports show a lock icon.

## Acceptance criteria

1. `POST /api/v1/exports` with `export_type: "pdf_executor"` and no
   `X-Reauth-Token` header returns 403.
2. `POST /api/v1/exports` with a valid reauth token from a `partner` user
   for `pdf_executor` returns 403.
3. A reauth token used twice is rejected on the second use (single-use enforcement).
4. Generated PDF summary contains account numbers as last-4 only ("••••4321").
5. Generated PDF executor contains full decrypted account numbers.
6. Generated Excel executor workbook contains a Transactions sheet with
   all transactions in the date range, filterable by auto-filter.
7. Export job status endpoint returns `complete` and `filename` after the
   ARQ task finishes.
8. Download endpoint streams the file with correct content-type and
   content-disposition headers.
9. Audit log contains an `export.generated` event after each successful export.
10. PDF executor includes an audit summary page with correct generation
    timestamp and generator name.
